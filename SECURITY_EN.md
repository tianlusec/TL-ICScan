# Security Policy

## Supported Versions

Currently, only security updates for the latest release are supported.

| Version | Supported          |
| ------- | ------------------ |
| > 0.5.x | :white_check_mark: |
| < 0.5.x | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please **DO NOT** disclose it publicly in GitHub Issues.

Please contact the maintenance team via the following methods:

- Send an email to: [security@example.com](mailto:security@example.com) (Please replace with actual email)
- Or contact via Keybase / PGP encryption (if available).

We will acknowledge receipt of your report within 48 hours and provide regular updates on the remediation progress.

### Scope

The security scope of this project is limited to:
- Data processing logic of `tianlu-intel-core`
- Collection logic of `tianlu_intel_collectors`
- Security of database storage

**Note**: The vulnerability intelligence (CVE data) collected by this project contains public vulnerability information by definition. This data does not constitute a security vulnerability of the project itself.
